## 「初めての Go 言語」を読んで Go を勉強している

一応分類上はフロントエンドエンジニアを名乗っているので、業務では TypeScript が主戦場なのですが、活動できる領域を広げるために、最近はサーバサイドも少しずつ勉強しています。

というかもともと Ruby から始めたんで、どちらかというとフロントエンドの方が後発ではあるんですが 😅

というわけで、[初めての Go 言語 ―他言語プログラマーのためのイディオマティック Go 実践ガイド](https://www.amazon.co.jp/%E5%88%9D%E3%82%81%E3%81%A6%E3%81%AEGo%E8%A8%80%E8%AA%9E-%E2%80%95%E4%BB%96%E8%A8%80%E8%AA%9E%E3%83%97%E3%83%AD%E3%82%B0%E3%83%A9%E3%83%9E%E3%83%BC%E3%81%AE%E3%81%9F%E3%82%81%E3%81%AE%E3%82%A4%E3%83%87%E3%82%A3%E3%82%AA%E3%83%9E%E3%83%86%E3%82%A3%E3%83%83%E3%82%AFGo%E5%AE%9F%E8%B7%B5%E3%82%AC%E3%82%A4%E3%83%89-Jon-Bodner/dp/4814400047)を読み始めました。

## 他言語プログラマーのための、というだけあって、説明が簡潔でわかりやすい

この書籍の原著もオライリーのサブスクで読めるのですが、元々のタイトルは「Learning Go」なんですよね。なので、副題は日本語版オリジナルな訳ですが、5 章まで読み終わった段階でも、「ああたしかにこのタイトルで納得」という内容です。

例えば制御構造の章で for 文が出てくるわけですが、他の言語を何か一つでもやっていれば for 文がどうやって動作するかは大体想像がつくわけで。そういった箇所には「想像どおり」とか「予想どおり」といった記述で済ませているので、他言語をやったことがある人にはとても読みやすいです。

## Go 言語の学習目標

仕事で扱っている Go のコードも触れるようにするところを 3 月入るまでにやりたいなと思っています。

が、だいたいそういうアバウトな目標はなーなーになって達成できないので、何個か溜まっている CLI ツール案を Go で作ってみるところを中間目標に置いています。

候補としてははてなブログからこのブログに移管するツールが作れると題材としてちょうどいいかなという感じです。

## とりあえず 5 章まで読んでみた時点での感想

5 章までで、変数、制御構造、関数、配列、スライス、マップ、構造体あたりを習得するわけですが（全然序盤）、慣れていないこともあって Go 言語の旨味を感じられていない...。

主戦場が TypeScript なこともあって、イメージ的にはもっとガッチガチに縛ってくれることを期待していたんですが、そういう感じのテイストではなく、シンプルに書けるってところがいいのかなあ、という印象です。

とりあえず本はめちゃめちゃわかりやすいので（ただし動かすサンプルは少ないので、適当に別の場所から補ってくる必要がある）、
このまま読み進めていきます。
