## 何を作ったの？

以下の4つの質問に答えると、それにあった[Spotify](http://d.hatena.ne.jp/keyword/Spotify)のプレイリストを作成してくれるnpmです（15曲入っています）。

1. 最近聴いた曲から1\~3曲
2. 曲の明るさ
3. アーティストの人気度
4. 曲の[BPM](http://d.hatena.ne.jp/keyword/BPM)

[recommendify - npm](https://www.npmjs.com/package/recommendify)

<img alt="f:id:ikmbear:20211017204037g:plain" src="https://cdn-ak.f.st-hatena.com/images/fotolife/i/ikmbear/20211017/20211017204037.gif" title="" height="624" width="1200" />

## どうして作ったの？

[Spotify](http://d.hatena.ne.jp/keyword/Spotify)のプレミアムプランに契約して、まあまあ曲は聴いているし、毎日使っているんですが、**ついつい同じ曲を選んでしまうんですよね**。  
自分で探しに行けばいいんですけど、[Spotify](http://d.hatena.ne.jp/keyword/Spotify)の検索画面ってジャンルを選ぶと人気の曲が出てくるので、結局最近聴いた曲ばっか出てきてしまう...。

そこで「いっそのことプログラムに勝手にプレイリストを作らせてしまえばいいのでは」と思い、本npmを作成するに至りました。

## どうやって使うの？

[recommendify - npm](https://www.npmjs.com/package/recommendify)にも記載がありますが、一応日本語でも記載しておきます。

※[Spotify](http://d.hatena.ne.jp/keyword/Spotify)アカウントを持っていることが前提になります。

### 実行

1. recommendifyをインストールする

```
% npm i -g recommendify
```

1. 以下のコマンドを実行し、クライアントIDとシークレットを入力して、[トーク](http://d.hatena.ne.jp/keyword/%A5%C8%A1%BC%A5%AF)ンの取得を行う

```
% recommendify settoken
```

<img alt="f:id:ikmbear:20211017204810p:plain" src="https://cdn-ak.f.st-hatena.com/images/fotolife/i/ikmbear/20211017/20211017204810.png" title="" height="116" width="670" />

1. これでrecommendifyが実行可能になります！プレイリストを作成しましょう！

```
% recommendify
```

## 工夫したところは？

曲をどうやって選出するかですね。

[Spotify](http://d.hatena.ne.jp/keyword/Spotify)のWebAPIでは結構いろいろなパラメータがあって、最初はそれらを全部選べるようにもしようかと思ったのですが、ランダムな曲を選ぶのにパラメータをいくつも選ぶのはいかがなものかと思いまして、現在の形に至りました。

ガチャを回すので4手くらいが限界かなあと。

あとは一応同じ選択肢を選んでもランダムになるように、曲にはシャッフルをかけています（一応[Spotify](http://d.hatena.ne.jp/keyword/Spotify)の[API](http://d.hatena.ne.jp/keyword/API)は同じパラメータだと同じ内容を返すように見えましたが、登録されている曲のリストが変われはその限りではないとも思っています）。

## 難しかったところは？

アクセス[トーク](http://d.hatena.ne.jp/keyword/%A5%C8%A1%BC%A5%AF)ンが切れた場合のリフレッシュ処理に少し苦労しました。

[Spotify](http://d.hatena.ne.jp/keyword/Spotify)のアクセス[トーク](http://d.hatena.ne.jp/keyword/%A5%C8%A1%BC%A5%AF)ンはたしか1時間くらいで切れてしまうのですが、切れるたびにエラーになってはストレスなので、confというnpmを使ってリフレッシュ[トーク](http://d.hatena.ne.jp/keyword/%A5%C8%A1%BC%A5%AF)ンをローカルに保存し、アクセス時に[トーク](http://d.hatena.ne.jp/keyword/%A5%C8%A1%BC%A5%AF)ンの有効期限が切れていれば、アクセス[トーク](http://d.hatena.ne.jp/keyword/%A5%C8%A1%BC%A5%AF)ンの再取得を行うようにしました。

```
const callApi = (func, argumentArray) => {  return (async function callSpotifyApi (retryCounter = 1) {    try {      return await func.apply(spotifyApi, argumentArray)    } catch (error) {      if (error.body.error.status === 401 && retryCounter) {        await refreshAccessToken()        return await callSpotifyApi(retryCounter - 1)      }      throw new Error(error)    }  }())}
```
