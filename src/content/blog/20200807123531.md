## [Vim](http://d.hatena.ne.jp/keyword/Vim)では`ii`コマンドで全選択できる？

[Google](http://d.hatena.ne.jp/keyword/Google)検索で「[vim](http://d.hatena.ne.jp/keyword/vim) 全選択」と検索すれば、こんな感じのコマンドが出てきます。

1. gg
2. Shift + v（行選択ビジュアルモードにしないと、最後の一行がうまく選択されないので注意）
3. G

<img alt="f:id:ikmbear:20200807101559g:plain" src="https://cdn-ak.f.st-hatena.com/images/fotolife/i/ikmbear/20200807/20200807101559.gif" title="" height="424" width="669" />

この方法が正攻法なのですが、へっぽこ[vimmer](http://d.hatena.ne.jp/keyword/vimmer)である私はキーボー[ドガ](http://d.hatena.ne.jp/keyword/%A5%C9%A5%AC)チャコンの末に別の方法で全選択する方法を見つけました（注意：結論としてはこれは全選択のコマンドではありません）！

<img alt="f:id:ikmbear:20200807102024g:plain" src="https://cdn-ak.f.st-hatena.com/images/fotolife/i/ikmbear/20200807/20200807102024.gif" title="" height="438" width="669" />

ビジュアルモードにした後、コマンド`ii`を入力すると、上図のような結果（全選択）になります。

「テキストオブジェクト「ii」を入力すると全選択できる？ 」  
「いやちょっと待てよ、テキストオブジェクトで「i」なんて聞いたことないぞ」  
そこで今回は`ii`すなわち「inner i?」の「i」とはなんぞやを調べてみました。

## `i`は「インデントオブジェクト」（VScodeVim）

まず同じテキストファイルに対して、シェル上で同じコマンドを入力しても全選択にはなりません（無効なキーを押した時のあの音がなります）。

ということは、この機能は[VSCode](http://d.hatena.ne.jp/keyword/VSCode)の[拡張機能](http://d.hatena.ne.jp/keyword/%B3%C8%C4%A5%B5%A1%C7%BD)「VSCodeVim」特有の機能ということになります。

そこで[拡張機能](http://d.hatena.ne.jp/keyword/%B3%C8%C4%A5%B5%A1%C7%BD)のページを確認してみると、「[vim](http://d.hatena.ne.jp/keyword/vim)-indent-object」なるものが。

<img alt="f:id:ikmbear:20200807102402p:plain" src="https://cdn-ak.f.st-hatena.com/images/fotolife/i/ikmbear/20200807/20200807102402.png" title="" height="604" width="1200" />

> Based on [vim](http://d.hatena.ne.jp/keyword/vim)-indent-object, it allows for treating blocks of code at the current indentation level as text objects. Useful in languages that don't use braces around statements(e.g [Python](http://d.hatena.ne.jp/keyword/Python))

要はカーソルの現在位置と同じインデントを、テキストオブジェクトとして扱うことができるということです（以降インデントオブジェクトと呼ぶ）。

（私は[Python](http://d.hatena.ne.jp/keyword/Python)触ったことないので初めて知りましたが）[Python](http://d.hatena.ne.jp/keyword/Python)のようにインデントでブロックを構成する言語には、このインデントオブジェクトはめちゃ便利ということらしいです。

```
a = 3if a == 5:    print "AAA"    # if文の対象    print "BBB"    # if文の対象print "CCC"        # if文の対象ではない
```

引用：[とほほのPython入門 - 構文 - とほほのWWW入門](http://www.tohoho-web.com/python/syntax.html)

## 実践！インデントオブジェクト

VSCodeVimの概要ページにはコマンドの説明もあるので、実際に試してみます（ビジュアルモードで試すので、オペレーターは指定しません）。

### `<operator> ii` ：カーソルの位置と同じインデントを指定

<img alt="f:id:ikmbear:20200807102640g:plain" src="https://cdn-ak.f.st-hatena.com/images/fotolife/i/ikmbear/20200807/20200807102640.gif" title="" height="442" width="669" />

冒頭でのべた「iiで全選択」は**「一番外側のインデントにカーソルをおいて、インデントオブジェクト`ii`を指定すれば、実質全選択」**ということでした。

（サンプルのアニメで上下移動も怪しいへっぽこ[vimmer](http://d.hatena.ne.jp/keyword/vimmer)だということがばれましたね...）

### テキストで再確認

テキストで各コマンドの範囲を確認すると、以下のようになります。

```
<div>  <p>vimvimvimvimvimvimvimvimvimvimvimvimvim</p>    <div>　<!--aiと、aIの上方向はここまで-->      <p>vimvimvimvimvimvimvimvimvimvimvimvimvim</p> <!--iiだとここまで-->      <p>vimvimvimvimvimvimvimvimvimvimvimvimvim</p>      <p>vimvimvimvimvimvimvimvimvimvimvimvimvim</p>      <p>vimvimvimvimvimvimvimvimvimvimvimvimvim</p> <!--ここでコマンドを実行-->    </div> <!--aIの下方向はここまで--></div>
```

## まとめ

- VSCodeVimでは、標準で「インデントオブジェクト」がサポートされている。
- 「インデントオブジェクト」：現在のカーソル位置のインデントを基準として、同じインデントの範囲を指定する。
- 一番外側のインデントにカーソルをおいてインデントオブジェクトを指定すると、実質全選択をすることができる。
